# Outer Witness - 项目指令集 (GEMINI.md)

本项目是一款仿造《星际拓荒》(Outer Wilds) 开发的 Unity 游戏，核心在于物理驱动的星系探索、无缝切换和动态天文系统。

## 核心原则 (Core Mandates)

1. **物理真实性优先**：所有天体运动和飞船航行必须基于物理力学。避免使用非物理的坐标位移（除特殊优化需求外）。
2. **大尺度坐标处理**：鉴于 Unity 在极远距离下的浮点数精度问题，星系系统设计必须考虑“漂浮原点”(Floating Origin) 或多层级坐标系方案。
3. **无缝体验**：禁止在行星间旅行时出现加载界面。所有关卡和天体应通过异步加载或视锥体裁剪进行管理。
4. **确定性模拟**：星系中的天体轨道（如开普勒轨道）应支持预计算或确定性更新，以确保游戏逻辑的同步。

## 工程规范 (Engineering Standards)

### C# 编码风格
- **命名规范**：PascalCase 用于类名和公共成员，camelCase 用于私有字段（加下划线前缀，如 `_gravityConstant`）。
- **组件化设计**：遵循“单一职责原则”。例如，将 `OrbitalBody`（轨道数据）与 `GravitySource`（引力逻辑）分离。
- **性能敏感**：在 `Update` 或物理循环中严禁使用 `GetComponent` 或 `GameObject.Find`。所有引用需在 `Awake` 或 `Start` 中缓存。

### Unity 实践
- **物理步长**：由于涉及高速航行，`Fixed Timestep` 需根据测试情况进行精细调整（建议 0.01s - 0.02s）。
- **材质与着色器**：考虑到多光源和星球大气，优先使用 URP (Universal Render Pipeline)，并保持 Shader 的性能高效。

## 当前开发阶段：星系系统 (Galaxy System) - 基础架构已建立

### 已实现模块
1. **OrbitSystem (核心管理器)**: 
   - 统一管理所有天体更新。
   - 采用深度优先排序（Depth Sorting），确保父天体先于子天体更新，彻底消除坐标抖动。
2. **OrbitBody (数据组件)**:
   - 采用参数化椭圆轨道：**长半轴 (a)** + **偏心率 (e)**。
   - 自动计算短半轴 (b)，支持轨道倾角 (Normal Axis)。
   - 提供编辑器实时 Gizmos 预览。

### 核心物理约束 (Orbit Rules)
- **参数唯一性**: `Eccentricity` 是轨道形状的唯一真值。禁止手动设置 `SemiMinorAxis`。
- **更新顺序**: `OrbitSystem` 必须在每帧统一计算位置，严禁在 `OrbitBody` 中编写位移逻辑。
- **坐标系**: 轨道计算在中心天体的局部平面进行，随后映射至世界坐标。

### 下一步计划
1. **自转系统 (Rotation System)**: 实现天体围绕自身轴心的旋转。
2. **漂浮原点 (Floating Origin)**: 处理大尺度空间下的坐标精度问题。
3. **开普勒第三定律适配**:（可选）根据轨道半径自动计算角速度。


## 资源管理
- 所有模型和材质放置在 `Assets/Models` 和 `Assets/Materials`。
- 场景文件放置在 `Assets/Scenes`，并按星系名称分子文件夹。
